# Git Workflow

## Commit Messages

Conventional commits enforced by commitlint + husky:

```
feat(native): add settings screen
fix(ui): resolve button variant styling
docs: update README for npm workspaces
chore(deps): update expo to 55.0.17
```

Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`,
`build`, `ci`, `chore`, `revert`

## Branch Strategy

- `main` — production-ready code
- Feature branches: `feat/feature-name`
- Bugfix branches: `fix/bug-description`

## Pre-commit Hooks

Husky runs on every commit:

- `npx lint-staged` — ESLint + Prettier on staged files
- `npx commitlint --edit` — validates commit message format

## Changesets

For versioning shared packages:

```bash
npm run changeset        # create a changeset
npm run changeset:version  # bump versions
npm run changeset:publish  # publish to npm
```

## Files to Never Commit

- `.env` files with real secrets
- `ios/` and `android/` directories (generated by prebuild)
- `node_modules/`
- `*.keystore` or `*.p12` signing files
- `google-services.json` with real credentials
